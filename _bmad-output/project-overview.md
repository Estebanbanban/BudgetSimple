# Budgetsimple - Project Overview

**Date:** 2025-12-20
**Type:** Web (static SPA)
**Architecture:** Local-first, browser-only

## Executive Summary

Budgetsimple is a single-page, local-first budgeting app that runs without a backend. Users import transactions and income from CSV (or enter manually), and the app computes dashboards (cashflow, budgets, month-over-month charts, top merchants) and planning tools (budgets + envelopes). Data stays on-device using IndexedDB and localStorage.

## Project Classification

- **Repository Type:** Single app (static site)
- **Project Type(s):** Browser SPA, local-first storage
- **Primary Language(s):** JavaScript, HTML, CSS
- **Architecture Pattern:** DOM-driven view rendering (no framework)

## Technology Stack Summary

- **Runtime:** Browser
- **UI:** `index.html` + `styles.css`
- **Logic:** `app.js` (no bundler)
- **Storage:** IndexedDB + localStorage
- **Charts:** Inline SVG rendering (generated by JS)

## Key Features

- Dashboard: summaries and charts over a selected date range
- CSV import for transactions + income (column mapping, dedupe support)
- Category system + auto-categorization rules (regex + exact-match memory)
- Monthly budgets per category with progress alerts
- Envelope savings goals with contributions and projected ETA
- Data export (summary CSV) and full local reset

## Architecture Highlights

- Single `main()` bootstraps config, storage, routing, event wiring, and first render.
- Data pipeline: load stored rows → compute aggregates (`summarize`) → render each view.
- Storage is abstracted behind a `store` object (IndexedDB-backed `DB`, with a localStorage fallback).

## Development Overview

### Prerequisites

- Any modern browser with IndexedDB enabled (fallback exists but IndexedDB is preferred).
